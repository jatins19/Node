API Documentation

login API

This API allows users to log in. You can use it for login with username, email, or phone number. However, the value must be passed in the username key.

Endpoint: /auth/login

Method: POST

cURL :- curl --location 'http://localhost:3000/login' \ --header 'Content-Type: application/x-www-form-urlencoded' \ --data-urlencode 'email=dheeraj1@mail.com' \ --data-urlencode 'password=12345678'

API Payload

{
    "username": "user@example.com", // or phone number, or username
    "password": "12345678"
}

API Response

{
    "status": "true",
    "message": "Login Successfully",
    "data": {
        // User Details
        "token":"jwt-token"
    }
}

Signup API

This API allows users to sign up. It is dynamic and can be used for signup by email, phone number, or username. You can also set a role by passing it in the parameters. If the user directly logs in after signup, the direct_key should be true.

Endpoint: /auth/signup

Method: POST

Accepted Keys:

  • email
  • password
  • direct
  • first_name
  • last_name
  • dob
  • address
  • phone_number
  • country_code
  • username :"if you have user name"

cURL :- curl --location 'http://localhost:3000/signup' \ --header 'Content-Type: application/x-www-form-urlencoded' \ --data-urlencode 'email=dheeraj1@mail.com' \ --data-urlencode 'password=12345678' \ --data-urlencode 'direct=true'

API Payload

{
    "email":"user@example.com", // or phone number, or username
    "password":"12345678"
    "direct":true // true if the user logs in directly after signup
}

API Response

{
    "status": "true",
    "message": "User registered successfully",
    "data": {
        // User Details
        "token": "jwt-token"
    }
}

Forgot Password API

This API allows users to request a password reset. You can use it with an email or phone number by passing the value in the username key. If both email and phone number are provided, it will send a reset link/code to both . If you want to process with OTP then You have to send otp in params.

Endpoint: /auth/forgot-password/:otp

Method: POST

cURL :- curl --location 'http://localhost:3000/auth/forgot-password/check' \ --header 'Content-Type: application/json' \ --data-raw '{ "username":"@gmail.com" }'

API Payload

{
    "username": "user@example.com", // or phone number
    // if both email and phone number are provided, both will receive reset instructions
}

API Response

{
    "success": true,
    "message": "Password reset instructions sent",
}

     //username required id OTP method

API Response

{

Verify OTP API

This API For Varify OTP on time of forgot password. Username field is required along with OTP.

Endpoint: /auth/verify-otp

Method: POST

cURL :- curl --location 'http://localhost:3000/auth/varify-otp' \ --header 'Content-Type: application/json' \ --data-raw '{ "otp": "037488", "username": "dheeraj1visionvivante@gmail.com" }'

API Payload

{
    "otp":  // OTP
    "username": "user@example.com"
}

API Response

{
    "status": true,
    "message": "OTP Varified"
}
